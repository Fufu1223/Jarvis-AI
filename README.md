* **NameError (未定义变量)**:
    * *现象*: 在终端直接输入中文 "你是谁"。
    * *原因*: 混淆了“在程序运行中输入数据”与“向 Python 解释器发送代码指令”。
    * *教训*: 只有在 `input()` 暂停时才能输入数据，否则 Python 会把文字当成变量名。
* **ModuleError (找不到入口)**:
    * *现象*: `can't find '__main__' module`。
    * *原因*: 误将 `app.py` 建成了文件夹。
    * *教训*: 文件 (File) 与 文件夹 (Folder) 是计算机眼中的不同物种。
* **PowerShell 权限**:

## 📅 Day 2: 给 AI 的记忆分类 (变量与数据类型)

### 🏆 核心掌握 (Mastery)

1.  **四大核心数据类型 (The Big 4)**:
    * `str` (字符串): 所有带引号的文字 (如 `"GPT-4"`)。**注意**: AI 的输入输出默认都是此类型。
    * `int` (整数): 用于计数、循环 (如 `10`)。
    * `float` (浮点数): 用于价格、精度参数 (如 `0.02`)。
    * `bool` (布尔值): 只有 `True`/`False`，用于逻辑开关。

2.  **Input 的欺骗性**:
    * `input()` 接收到的任何内容（哪怕你输入的是数字 100），在 Python 眼里全是 **字符串** (`"100"`)。
    * **关键动作**: 必须使用 `int()` 进行“变身”才能进行数学计算。

3.  **Prompt Engineering 核心技能**:
    * **f-string (格式化字符串)**: 告别丑陋的 `+` 号拼接。
    * **语法**: `f"你好，我是{name}"`。这是编写动态 AI 提示词模板的行业标准方式。

### 💀 错题本 (Bug Log)

* **TypeError (类型错误)**:
    * *代码*: `print("明年" + 26 + "岁")`
    * *原因*: 试图强行把 **文字(str)** 和 **数字(int)** 用 `+` 号拼在一起。
    * *修正*: 使用 `str(26)` 转换，或者直接用 f-string。

* **ValueError (值错误)**:
    * *代码*: `int("哈哈")`
    * *原因*: 试图把非数字的文字强转为整数。
    * *教训*: "Garbage In, Garbage Out"。必须确保输入的是数字格式。
